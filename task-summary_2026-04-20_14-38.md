# 任务摘要 2026-04-20 14:27-14:38 — 密码登录功能实施与Bug修复

## 目标
为 trading-scraper 工具添加密码锁屏功能（密码 881199）

## 完成内容

### 1. 后端（proxy-server.js）
- 新增 `POST /unlock` 端点：独立密码验证，不调ASP，纯本地比对 `APP_PASS`
- 新增 `GET /lock` 端点：重置 `appLoggedIn = false`
- `appLoggedIn` 默认 `false`，需解锁后才能访问 `/fetch` 等数据端点
- `/status` 返回 `appReady: appLoggedIn` 供前端判断锁屏状态

### 2. 前端（index.html）
- 全屏锁屏界面（#screenLock）：深色背景 + 居中卡片 + 密码输入框 + 解锁按钮
- Header 右上角加 🔒 锁屏按钮
- 解锁流程：输入密码 → /unlock 验证 → autoLogin() ASP登录 → doFetch() 抓数据
- 刷新页面需重新输入密码（安全优先）

### 3. Bug修复
- **Bug1**：`/unlock` 和 `/lock` 路由用了未定义变量 `method`（应为 `req.method`），导致500错误
- **Bug2**：解锁后直接调 doFetch() 但未做 ASP 登录，roomCookie 为空导致抓取失败
  - 修复：doUnlock() 成功后先 await autoLogin() 再 await doFetch()
  - init() 里 appReady=true 时也检查 loggedIn，未登录则先 autoLogin()
